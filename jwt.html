🔐 How JWT Signing & Verification Really Works

1️⃣ What JWT Actually Is

JWT (JSON Web Token) is a digitally signed message format. It is NOT encrypted by default. Anyone can read the payload.

Security comes from the cryptographic signature which guarantees:

  • Integrity – data was not modified
  • Authenticity – issuer really created it

2️⃣ JWT Structure

HEADER.PAYLOAD.SIGNATURE

Each section is Base64URL encoded.

3️⃣ Header Example

{
  "alg": "RS256",
  "typ": "JWT"
}
  • RS256 = RSA + SHA-256
  • Asymmetric cryptography
  • Private key signs
  • Public key verifies

4️⃣ Payload Example

{
  "sub": "123456",
  "role": "admin",
  "iat": 1710000000,
  "exp": 1710003600
}

Payload is readable. It is protected from tampering — not hidden.

5️⃣ How Signature Is Formed

Step 1 — Encode

encodedHeader  = Base64UrlEncode(header)
encodedPayload = Base64UrlEncode(payload)

Step 2 — Create Signing Input

signingInput = encodedHeader + "." + encodedPayload

Step 3 — Hash

hash = SHA256(signingInput)

Step 4 — Sign With Private Key

signature = RSA_sign(hash, privateKey)

Step 5 — Encode Signature

encodedSignature = Base64UrlEncode(signature)

Final JWT:

encodedHeader.encodedPayload.encodedSignature

6️⃣ Why The Private Key Is NOT Inside The Signature

The private key is never embedded in the token. It is only used mathematically to create the signature.

  • Private key = secret stamping tool
  • Signature = wax seal impression

You cannot reconstruct the private key from the signature. RSA signing is a one-way mathematical operation: 

signature = hash^d mod n
  • d = private exponent (secret)
  • n = modulus

Public key contains (e, n). It cannot derive d.

7️⃣ How Verification Works

Receiver performs:

recomputedHash = SHA256(signingInput)
verifiedHash   = RSA_verify(signature, publicKey)

Internally:

hash = signature^e mod n

If recomputed hash equals verified hash → valid. If even 1 character changes → verification fails.

8️⃣ King Stamp Analogy

  • The king writes decree (payload)
  • He presses royal ring (private key)
  • Wax seal = signature

Citizens:

  • Have official drawing of ring (public key)
  • Compare seal pattern

The ring never leaves the king. The seal travels.

9️⃣ Security Properties

  • Private key stays only on issuer server
  • Public key can be shared with many services
  • Signature cannot reveal private key
  • Tampering breaks validation immediately

🏆 Important Clarification

RS256 provides integrity and authenticity. It does NOT provide confidentiality.

If you need secrecy:

  • Use HTTPS (always required)
  • Or use encrypted JWT (JWE)

JWT signature = mathematical proof of origin and integrity. Not encryption.
